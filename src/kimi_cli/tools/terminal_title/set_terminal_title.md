Set the terminal tab title to a short task name.

**MANDATORY**: Call this tool at the start of every new task. The user uses the tab title to identify which task each terminal tab is working on, especially when multiple tabs are open. If you skip this call, the tab title will remain generic and the user will lose track of which tab is which.

Call this tool when:
- The user assigns a new task
- You start a significant new phase that deserves its own title

Keep the title to 5 words or fewer — it must fit in a terminal tab.

Examples:
- "Refactor auth module"
- "Fix login bug"
- "Add test coverage"
- "Investigate build failure"

Do not call this tool repeatedly for the same task. One call per task is enough.
